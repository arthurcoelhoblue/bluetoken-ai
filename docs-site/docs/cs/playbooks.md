# Playbooks — Automação para Ações de CS

Um **Playbook** é um roteiro de ações automatizadas, acionado por um gatilho específico.

## Exemplo: Recuperação de NPS Detrator

1. **Gatilho**: Cliente responde NPS com nota 4
2. **Ação 1** (Automática): Sistema cria Incidência de gravidade "Alta"
3. **Ação 2** (Automática): Tarefa criada para o CSM: "Ligar para o cliente [Nome]"
4. **Ação 3** (Automática): E-mail interno enviado para o gestor de CS

:::tip Consistência
Os Playbooks garantem que a equipe siga as melhores práticas e que nenhuma situação crítica seja esquecida.
:::
